Methods & Results

Regarding data wrangling, we need to match the players and sessions dataset using hashedEmail, which exists in both datasets. We then need to aggregate the total number of sessions per player in a new variable (total_sessions), and also calculate the average session duration per player. Using the mutate function, we can create a new variable: avg_session_length, which would be computed by played_hours / total_sessions. We can then compare the played_hours and avg_session_length variables against experience, age, and gender to determine which characteristics are the strongest predictors of the amount of data a player contributes.

In order to do this we need to perform the following:

  • Store our players data frame in the tidy_players variable

  • Store our sessions data frame in the tidy_sessions variable, removing redundant data from the sessions data set

Wrangling of the sessions data set:

Again, the data is mostly tidy from the nature of this project. Although, there are multiple repetitions in the hashedEmail column, each row represents a different observation (unique session). Removing the original_start_time and original_end_time columns allows us to avoid redundancy; we only need start_time and end_time to calculate the total session time. We have also converted it to POSIXct format (dttm) from character format to simplify calculations.

tidy_sessions <- sessions |>

    mutate(end_time = as.POSIXct(end_time, format="%d/%m/%Y %H:%M", tz="PTC"))
